    En la mayoría de los casos, un URL simple se reconoce como tal y se vincula automáticamente: 

    Visit https://area51.stackexchange.com/ regularly!
Use angle brackets to force linking: Have you seen <https://superuser.com>?

    Para crear enlaces más elegantes, utiliza Markdown: 

    Here's [a link](http://www.example.com/)! And a reference-style link to [a panda][1].
References don't have to be [numbers][question].

 [1]: https://notfound.stackexchange.com/
 [question]: https://english.stackexchange.com/questions/11481

    Puedes añadir información sobre herramientas a estos enlaces:

    Click [here](https://diy.stackexchange.com "this text appears when you mouse over")!
This works with [reference links][blog] as well.

 [blog]: https://stackoverflow.blog/ "click here for updates"

    La palabra entre corchetes es el texto alternativo, que se muestra si el navegador no puede mostrar la imagen. Asegúrate de incluir un texto alternativo con sentido para el software de lectura en pantalla.

    Asegúrate de utilizar estilo de texto con moderación, solo cuando facilite la lectura.

    *This is italicized*, and so
is _this_.

**This is bold**, just like __this__.

You can ***combine*** them
if you ___really have to___.

    Para separar el texto en secciones, puedes utilizar encabezados: 

    A Large Header
==============

Smaller Subheader
-----------------

    Utiliza marcas hash si necesitas varios niveles de encabezados:

    # Encabezado 1 #
## Encabezado 2 ##
### Encabezado 3 ###

    Se pueden crear tanto listas con viñetas como numeradas:

    - Use a minus sign for a bullet
+ Or plus sign
* Or an asterisk

1. Numbered lists are easy
2. Markdown keeps track of
   the numbers for you
7. So this will be item 3.

    1. Lists in a list item:
    - Indented four spaces.
        * indented eight spaces.
    - Four spaces again.
2.  You can have multiple
    paragraphs in a list items.
 
    Just be sure to indent.

    > Create a blockquote by
> prepending “>” to each line.
>
> Other formatting also works here, e.g.
>
> 1. Lists or
> 2. Headings:
>
> ## Quoted Heading ##

    Incluso puedes colocar citas en bloque dentro de citas en bloque: 

    > A standard blockquote is indented
> > A nested blockquote is indented more
> > > > You can nest to any depth.

    Para crear bloques de código u otro texto con formato previo, aplica una sangría de cuatro espacios: 

        This will be displayed in a monospaced font. The first four spaces
    will be stripped off, but all other whitespace will be preserved.
    
    Markdown and HTML are turned off in code blocks:
    <i>This is not italic</i>, and [this is not a link](https://example.com)

    Para crear un intervalo de código insertado, y no un bloque, utiliza acentos graves: 

    El carácter `$` es un método abreviado para `window.jQuery`.

    Si deseas incluir un bloque con formato previo en una lista, aplica una sangría de ocho espacios: 

    1. This is normal text.
2. So is this, but now follows a code block:
 
        Skip a line and indent eight spaces.
        That's four spaces for the list
        and four to trigger the code block.

    Si necesitas hacer algo que el Markdown no admite, utiliza HTML. Ten en cuenta que solo admitimos un subconjunto de etiquetas HTML muy limitado. 

    El tipo de humor de tachado es <strike>gracioso</strike>.

    Markdown es lo suficientemente inteligente como para no arruinar el HTML de nivel de intervalo: 

    <b>El Markdown funciona *correctamente* aquí.</b>

    Los elementos de bloque HTML tienen algunas restricciones:

    1. Se deben separar del texto que las rodea con líneas en blanco.
    2. No se debe aplicar sangría a las etiquetas iniciales y finales del bloque más alejado.
    3. No se puede utilizar Markdown en bloques HTML.

    <pre>
    <em>No</em> puedes usar Markdown aquí.
</pre>

    ► usa saltos de línea para separar párrafos

    ► para añadir un salto de línea, coloca 2 espacios al final

    ► _cursiva_ o **negrita**

    ► indenta el código con 4 espacios

    ► el acento grave crea un escape `de _esta_ forma`

    ► crea un bloque de cita poniendo > al comienzo de la línea

    Cómo etiquetar back

    Una etiqueta es una palabra clave que categoriza tu pregunta con otras preguntas similares. Escoje una o más (hasta 5) etiquetas que ayudarán a quienes responden a encontrar e interpretar tu pregunta.

    ► completa la frase: mi pregunta es acerca de…

    ► usa etiquetas que describan conceptos o elementos que sean esenciales, y no de importancia secundaria, para tu pregunta

    ► tiende a utilizar las etiquetas que ya existen y son populares

    ► lee la descripción que aparece debajo de cada etiqueta

    Si tu pregunta es principalmente acerca de un tema para el cual no encuentras una etiqueta:

    ► combina múltiples palabras en una-sola-palabra usando guiones (ejemplo: android-studio), hasta un máximo de 35 caracteres

    ► crear nuevas etiquetas es un privilegio. Si aún no puedes crear la etiqueta que necesitas, entonces publica la pregunta sin esa etiqueta, y solicita que la comunidad la cree
